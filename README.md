# VaultForge

**Drop a PDF or EPUB. Get an Obsidian vault.**

[![VaultForge Screenshot](https://raw.githubusercontent.com/julianhilgemann/vaultforge/refs/heads/main/vaultforge_main_screenshot_03042026.png)](https://julianhilgemann.github.io/vaultforge/)

🔗 **[Live Demo → julianhilgemann.github.io/vaultforge](julianhilgemann.github.io/obsidian_claude_vault_forge_app/)**

---

VaultForge extracts knowledge from documents and transforms it into a structured, interconnected [Obsidian](https://obsidian.md) vault — atomic notes with `[[wikilinks]]`, concept tags, and a live knowledge graph. No install. No backend. No build step. One HTML file.

---

## What it does

1. **Drop a file** — PDF or EPUB
2. **Select sections** — VaultForge detects chapters and logical sections automatically
3. **Configure** — choose note style, notes per section, vault name
4. **Generate** — each section is sent to Claude via the Anthropic API; notes come back as structured markdown
5. **Preview** — explore the knowledge graph, browse notes, read raw markdown
6. **Download** — get a `.zip` ready to open directly in Obsidian

---

## Note styles

| Style | Description |
|---|---|
| **Zettelkasten** | One concept per note, heavily interlinked, self-contained |
| **Summary** | Thematic notes capturing key arguments per section |
| **Academic** | Structured notes with definitions, evidence, and implications |

All styles produce frontmatter-compatible markdown with `[[wikilinks]]` that resolve automatically in Obsidian.

## Vault structure

```
YourVault/
├── notes/
│   ├── Note Title One.md
│   ├── Note Title Two.md
│   └── ...
├── _index.md       ← map of content: hub concepts + tag clusters
└── README.md
```

Open the folder as a vault in Obsidian. The graph view will light up immediately with all wikilinks resolved.

## Your API key is safe

VaultForge runs **entirely in your browser**. There is no server, no backend, no database.

- Your document is parsed locally using [pdf.js](https://mozilla.github.io/pdf.js/) and [JSZip](https://stuk.github.io/jszip/) — **it never leaves your machine**
- Your API key is held in JavaScript memory for the duration of your session — **it is never stored, never logged, never transmitted anywhere except directly to `api.anthropic.com`**
- API calls go from your browser straight to Anthropic's servers over HTTPS — **no third party ever sees your key or your content**
- When you close the tab, everything is gone — notes, key, document

You can verify this yourself: open DevTools → Network tab → every request goes to `api.anthropic.com` and nowhere else.

Get your API key at [console.anthropic.com](https://console.anthropic.com).

## Deploy to GitHub Pages

```bash
# 1. Clone or fork this repo
git clone https://github.com/yourusername/vaultforge

# 2. That's it — index.html is the entire app
```

Then in your repo: **Settings → Pages → Deploy from branch → main → / (root)**

Your app will be live at `https://yourusername.github.io/vaultforge` in about 60 seconds. No build pipeline. No Node. No config files.

## How document parsing works

**PDF** — uses pdf.js to extract text with font-size metadata. Headings are detected by comparing font size to the document's body text size. If no typographic hierarchy is found, the document is split into equal word-count chunks automatically.

**EPUB** — unzipped client-side with JSZip. Reads `META-INF/container.xml` to locate the OPF manifest, then follows the spine to extract chapters in reading order.

Each section is sent to the LLM as an independent call with a maximum of ~2,800 words. Long chapters are trimmed to this window. Cross-section wikilinks are generated by concept name and resolve in Obsidian when titles match.

Zero npm dependencies. Zero build tools. Zero frameworks.

---

## Limitations

- Sections longer than ~2,800 words are trimmed before being sent to the LLM. Dense chapters may lose their tail.
- Many sequential API calls on a large book may hit Anthropic rate limits on lower-tier keys.
- Older or unconventionally packaged EPUBs (some Project Gutenberg files) may not parse correctly.
- The knowledge graph preview is session-only. Download the vault before closing the tab — there is no recovery.
